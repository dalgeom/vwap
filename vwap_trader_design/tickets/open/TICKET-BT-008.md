## 목적

ATR-Relative Rest(ARR) 레짐 조건이 BTC/ETH 에서 일평균 충분한 발동 빈도를 보이는지 실증.
PASS 시 ARR을 레짐 필터로 채택, FAIL 시 폐기.

---

## Regime 조건 (명세 그대로 구현, 임의 수정 금지)

```
ATR14_1H / mean(ATR14_1H, 20봉) < 1.0
AND abs(EMA9 - EMA20) / close < 0.003
```

- `ATR14_1H`: 1시간봉 기준 ATR(14)
- `mean(ATR14_1H, 20봉)`: 직전 20봉 ATR14 단순 평균
- `EMA9`, `EMA20`: 1시간봉 기준 각각 9, 20주기 EMA

---

## PASS / FAIL 기준

| 판정 | 조건 |
|---|---|
| **PASS** | 일평균 ARR regime 발동 ≥ 6건 |
| **FAIL** | 일평균 < 4건 |
| **경계** | 4건 이상 6건 미만 → 조건 1~5 실측 통과율 병행 보고 필수 |

---

## 추가 보고 의무 (C-22-3)

- **조건 1 비회귀 비율**: ARR regime 발동 중 ±2σ 이탈(조건 1) 발생 후 4시간(4H) 이내 가격 미회귀 비율
  - ≥ 50% 시: **즉시 의장(Dev-PM) 보고** → ARR 채택 불가 트리거 발동
  - < 50% 시: 수치 그대로 보고서에 포함

## 병목 분석 의무

보고서에 다음 항목 명시:
- ATR 조건(`ATR14_1H / mean < 1.0`)이 단독으로 발동 차단한 비율
- EMA 조건(`abs(EMA9 - EMA20) / close < 0.003`)이 단독으로 발동 차단한 비율
- 양 조건 동시 차단 비율
